## 12 类收尾验收提示词

### 基础阶段（必选）
1. **代码语法类** — TypeScript 编译/ESLint/React Console 警告/JSDoc 覆盖率 >90%/代码规范统一/无循环依赖
2. **功能完整逻辑类** — 核心功能完整/业务逻辑正确/性能达标/UX 流畅/安全合规/兼容性
3. **测试用例类** — 单元 >80%/集成 >70%/E2E 覆盖所有流程/性能测试/安全测试/兼容测试
4. **组件测试类** — 所有组件测试/覆盖率 >85%/交互+状态+样式+无障碍测试
5. **单元框架类** — Vitest 配置/工具函数/覆盖率配置/CI/CD 集成
6. **闭环验证类** — 全功能+全质量+全文档+全部署+全兼容+全性能验证

### 进阶阶段（推荐）
7. **统一化类** — 设计语言/代码规范/文档风格/交互体验完全统一
8. **现状审核分析建议类** — 代码质量/功能完整性/UX/技术架构/项目管理审核 + 改进路线图
9. **MVP功能拓展类** — 核心增强/UX 提升/集成扩展/智能化/生态建设

### 高级阶段（可选）
10. **高级功能完善类** — AI 智能化/自动化/集成化/协作化/可视化/个性化
11. **性能优化类** — 首屏 <2s/切换 <100ms/内存 <500MB/CPU <30%/60fps
12. **安全加固类** — 无高/中危漏洞/安全评分 >90/通过渗透测试/符合 OWASP

### 最终验收评分
- 代码质量 >= 90
- 功能完整性 >= 90
- 测试覆盖率 >= 80
- 性能 >= 90
- 安全性 >= 90
- 文档完整性 >= 95
- 兼容性 >= 90
