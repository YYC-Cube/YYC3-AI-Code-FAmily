## 🎯 **设置页面功能需求提示词**
## 🔗 **与项目现状集成说明**

设置页面的实现需与现有项目架构深度整合，确保数据统一和功能互通：

1. **状态管理统一**  
   采用项目已规划或建议的全局状态方案（如 React Context + useReducer 或 Zustand），将设置配置提升至应用顶层（例如 `Chat.tsx` 或专门的 `SettingsProvider`）。避免在每个组件内部维护独立状态，保证主题、模型、快捷键等配置在全局范围内即时生效。

2. **数据持久化互通**  
   复用现有的持久化机制（如 `useChatPersistence` 中使用的 localStorage 或 Supabase），将用户偏好、API密钥、自定义规则等设置存入同一存储空间。确保重启应用后配置自动加载，并与聊天历史、项目规则等数据保持隔离但可交叉引用（例如根据当前模型切换自动调整对话行为）。

3. **模块间数据流打通**  
   - **模型配置**：保存后应直接影响 `ChatContainer` 中调用的 AI 服务商和模型参数。
   - **MCP 连接**：配置的协议端点需被工作流引擎、插件系统等模块读取，用于执行实际工具调用。
   - **规则与技能**：个人规则和项目规则需在智能体对话时动态注入系统提示词或约束条件，实现配置即生效。
   - **快捷键与通知**：设置应立即覆盖 VS Code 风格快捷键映射，并控制全局通知行为（横幅、声音等）。

4. **增量导入与覆盖策略**  
   “导入配置”功能需与现有数据格式兼容，导入前应深度合并或按用户选择覆盖，同时提供版本校验或冲突提示，防止破坏已有数据。

5. **项目级配置识别**  
   对于项目规则、项目级 MCP 等设置，需与工作区识别逻辑结合（基于当前打开的项目路径），动态加载 `.ai/` 目录下的配置文件，并与全局设置合并优先级。

6. **扩展性预留**  
   设置数据结构应设计为可扩展的键值对或模块化对象，方便未来添加新的服务商、插件或技能时无需重构现有状态管理逻辑。



请基于以下功能点设计或开发设置页面，**无需UI布局描述**，仅需确保所有列出的功能模块和选项被正确实现：

---

### 🔍 **搜索**
- 全局搜索功能，允许用户快速定位设置项。

---

### 👤 **账号**
- 个人信息编辑模块（包含头像、用户名、邮箱等基础信息）。

---

### ⚙️ **通用**
- **基础设置**：包含常规选项。
- **主题**：提供主题选择（如深色/浅色/跟随系统等）。
- **语言**：支持简体中文及其他语言选择。
- **偏好设置**：
  - **编辑器设置**：可配置字体、word wrap、窗口行为等。
  - **快捷键设置**：支持自定义AI操作的快捷键，提供VS Code风格的快捷键方案。
  - **导入配置**：允许导入VS Code或Cursor的插件、设置、代码片段、快捷键，覆盖当前配置（不可恢复）。
- **本地链接默认打开方式**：选择系统浏览器或内置浏览器。
- **Markdown文件默认打开方式**：选择代码编辑器或其他方式。
- **开发环境**：
  - **Node.js版本管理**：可添加/选择Node.js版本。

---

### 🧠 **智能体**
- **自定义智能体**：管理用户创建的智能体。
- **内置智能体**：查看和配置系统预置的智能体。

---

### 🔌 **MCP**
- **MCP列表**：支持添加新的MCP，选项包括从市场添加或手动添加。
- **项目级MCP开关**：允许自动加载项目根目录下 `.ai/mcp.json` 的配置。

---

### 🤖 **模型**
- **服务商**：下拉选择模型服务商（如智谱GLM等）。
- **模型**：选择具体模型。
- **API密钥**：输入框，并提供“获取API密钥”的跳转链接（直达服务商API获取页面）。
- **添加模型**：按钮用于新增模型配置。

---

### 📚 **上下文**
- **代码索引管理**：显示工作区索引构建进度/状态，提供刷新和删除索引的操作。
- **忽略文件**：配置额外的忽略文件规则（补充`.gitignore`），用于控制索引范围。
- **文档集**：支持添加文档集（通过URL或本地上传），作为AI问答的上下文来源。

---

### 💬 **对话流**
- **待办清单**：开关控制智能体是否使用待办清单跟踪任务进度。
- **对话流节点自动折叠**：开关控制已完成任务的对话节点是否自动折叠并生成摘要。
- **自动修复代码规范问题**：
  - **自动修复开关**：自动修复对话中检测到的代码问题。
- **智能体主动提问**：开关控制智能体是否在任务不明确时主动提问。
- **代码审查**：
  - **审查范围**：下拉选项（如“无需审查”、“审查全部”等）。
  - **审查后跳转**：开关控制审查后是否自动跳转到下一处变更。
- **自动运行**：
  - **自动运行MCP**：开关控制使用智能体时是否自动运行MCP工具。
  - **命令运行方式**：选择“沙箱运行”（支持白名单）或直接执行；白名单命令输入框。
- **任务状态通知**：
  - 通知方式开关：横幅、声音、菜单栏。
  - **音量设置**：滑块调节提示音量。
  - **提示音**：任务完成、等待操作、异常打断的提示音选择（每个可试听）。

---

### 📜 **规则和技能**
- **导入设置**：
  - 开关：将 `AGENTS.md` 包含在上下文中。
  - 开关：将 `CLAUDE.md` 和 `CLAUDE.local.md` 包含在上下文中。
- **个人规则**：创建和管理用户自定义规则，AI将在对话中遵循。
- **项目规则**：创建仅对当前项目生效的规则。
- **技能**：
  - 分类标签：全局 / 项目。
  - **全局技能**：在所有会话和项目中生效，支持创建新技能。
  - **项目技能**：仅当前项目生效，支持创建新技能。

---

**备注**：以上为设置页面的完整功能需求，实现时应确保各模块清晰独立，配置项可保存并生效。所有开关、下拉、输入框等交互元素需提供即时反馈。
